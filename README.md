# BookCatalog API

**Дисциплина:** Технологии разработки приложений на базе фреймворков  

---

## 📌 Описание проекта

Веб-служба для управления каталогом книг на ASP.NET Core 8. Реализует три точки доступа, хранение в памяти, валидацию данных и конвейер обработки запросов из трех middleware-компонентов.

---

## 🏗 Архитектурная идея

Проект построен на основе **конвейера обработки запросов (middleware)** в ASP.NET Core. Это позволяет разделить сквозные задачи и бизнес-логику:

| Middleware | Задача |
|------------|--------|
| `ErrorHandlingMiddleware` | Ловит все исключения, возвращает JSON с RequestId |
| `RequestResponseLoggingMiddleware` | Логирует входящие запросы и исходящие ответы |
| `RequestTimingMiddleware` | Измеряет время выполнения, добавляет заголовок |

### Почему это аккуратный каркас?

1. **Разделение ответственности** — каждый компонент делает одну вещь
2. **Переиспользуемость** — middleware можно использовать в любом проекте
3. **Тестируемость** — бизнес-логика не зависит от инфраструктуры
4. **Расширяемость** — новый функционал добавляется без изменения существующего кода

## 🚀 Инструкция запуска

### Требования
- .NET SDK 8.0 или выше

### Шаги

```bash
# 1. Клонировать репозиторий
git clone https://github.com/Dolfi-er/MiniWeb.git
cd MiniWeb

# 2. Запустить приложение
dotnet run


### Ручные тесты
Выполните команды в отдельном терминале после запуска приложения.

1. Создание книги (POST)
bash
curl -X POST http://localhost:5244/api/books -H "Content-Type: application/json" -d "{\"title\":\"Война и мир\",\"author\":\"Лев Толстой\",\"year\":1869,\"pageCount\":1225}"
Ожидаемый ответ: 201 Created с JSON созданной книги

2. Получение всех книг (GET)
bash
curl http://localhost:5244/api/books
Ожидаемый ответ: 200 OK со списком книг

3. Получение книги по ID (GET)
bash
curl http://localhost:5244/api/books/1
Ожидаемый ответ: 200 OK с книгой

4. Запрос несуществующей книги (404)
bash
curl http://localhost:5244/api/books/9999
Ожидаемый ответ: 404 Not Found

5. Проверка валидации — пустое название
bash
curl -X POST http://localhost:5244/api/books -H "Content-Type: application/json" -d "{\"title\":\"\",\"author\":\"Толстой\",\"year\":1869,\"pageCount\":1225}"
Ожидаемый ответ: 400 Bad Request с ошибкой "Название книги обязательно"

6. Проверка валидации — отрицательный год
bash
curl -X POST http://localhost:5244/api/books -H "Content-Type: application/json" -d "{\"title\":\"Книга\",\"author\":\"Автор\",\"year\":-100,\"pageCount\":100}"
Ожидаемый ответ: 400 Bad Request с ошибкой "Год издания должен быть между 0 и 2100"

7. Проверка валидации — отрицательные страницы
bash
curl -X POST http://localhost:5244/api/books -H "Content-Type: application/json" -d "{\"title\":\"Книга\",\"author\":\"Автор\",\"year\":2020,\"pageCount\":-50}"
Ожидаемый ответ: 400 Bad Request с ошибкой "Количество страниц не может быть отрицательным"

8. Проверка заголовка с временем выполнения
bash
curl -I http://localhost:5244/api/books
Ожидаемый ответ: должен присутствовать заголовок X-Response-Time-Ms
